## Explicit misalignments

**1. Stale version constraints on spec-runner (main finding of this run).**
`spec-runner` has reached version **2.9.0**, but consumers still pin minimums from the 0.1.x era:

- arbiter: `spec-runner >=0.1.1`
- atp-platform: `spec-runner >=0.1.4`

That's ~two majors behind in the declared compatibility contract. Formally `>=0.1.x` accepts 2.9.0 too, but such "soft" pins (a) don't reflect the actually-required version, (b) mask possible breaking changes between 0.1 and 2.9. Worth raising to a real range (e.g. `>=2.9,<3`) and locking it in CI.

**2. proctor: integration nearly complete, two loose ends left.**
The `proctor-a вҶ’ proctor` rename is done, a real `proctor` repo exists, and dispatcher's collector detects it (`config/proctor.yaml` present вҖ” verified). Remaining:

- The collector reads `data/state.db`, but the `proctor` repo has no such file вҶ’ runtime will warn "data/state.db missing or empty". Either the collector's DB path is stale, or proctor hasn't produced state yet. Needs reconciling.
- The name `proctor-a` was deliberately kept as an observability service-id (enum in arbiter's log schema, documented in ADR 2026-07-07 вҖ” `authored/decisions/2026-07-07-adr-proctor-identity.md`). That's an intentional dual-namespace now, not debt вҖ” but the collision with the new `proctor` repo can confuse readers; worth documenting once in dispatcher's README.

**3. steward вҖ” a claimed link without code.** README promises governance "over spec-runner/Maestro", but the graph shows zero edges. Either wire up real dependencies/contracts, or explicitly keep it in DRAFT status.

**4. robin-runtime not wired up.** M0 skeleton, deps commented out, `uv.lock` not committed.

**5. atp-platform вҖ” code-vs-docs drift (from prograph's drift findings).** prograph recorded 8 discrepancies in atp-platform: 7 MCP tools in `atp-dashboard/mcp/tools.py` (`ping`, `mcp_list_tournaments`, `mcp_get_tournament`, `mcp_get_history`, `mcp_leave_tournament`, `mcp_cancel_tournament`, `_join_tournament_mcp`) are implemented but not listed in the intent docs; conversely, `list_tournaments`, declared in spec `2026-04-15-el-farol-tournament-design.md:355`, has no implementation (`missing`). Docs/spec should be synced with code.

**What's NOT wrong (checked):** no drift in benchmark/observability/spec-runner schemas (all copies single-hash), no MCP-protocol mismatch between MaestroвҶ”arbiter (1.1.0), no divergence from upstream, no real uncommitted changes. Core contract discipline remains healthy. вң…
