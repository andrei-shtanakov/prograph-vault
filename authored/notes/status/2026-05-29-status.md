## New projects — require a registry decision

### `prograph` 🆕 — cross-project structure mapper (Rust + Python, v1.3 / M11)
- **What it does (README):** discovers how independent projects in a workspace are linked (package deps, shared contracts, MCP calls); serves the graph to humans (browser UI + REST `:7700`) and to agents (MCP stdio `prograph mcp`). Export to Markdown / Obsidian.
- **M11 (drift detection):** on each `index` it compares declarations (`README.md`/`TODO.md`/specs) against reality; writes findings `missing` / `extra` / `stale_todo`. CLI `prograph drift`, MCP `find_drifts`, REST `GET /api/drifts`.
- **⚠️ Strategically:** prograph duplicates/automates exactly the work being done by hand in `COWORK_CONTEXT.md` (integration map + contract drift). Worth deciding: make prograph the source of truth for the ecosystem map, or keep it as an experiment.

### `appgraph` 🆕 — graph-based application composition (Python, prototype)
- **README:** typed nodes → DAG → topological scheduler (optionally in parallel). Self-assessment: *"early prototype, not wired into the orchestrator ecosystem yet"*.
- **⚠️ Overlap:** the README directly admits a conceptual overlap with the Maestro DAG scheduler. A candidate for scope delimitation or absorption.

### `prograph-vault` 🆕 — Obsidian store, the sink for the prograph export (not a separate repo).

> All three are absent from the COWORK_CONTEXT registry. Indirect confirmation of their impact: the only atp-platform commit of the week (`3a7234a`) is *"configure prograph exclude for non-project sub-dirs"*, i.e. the new directories already required configuration changes in a neighbor.
