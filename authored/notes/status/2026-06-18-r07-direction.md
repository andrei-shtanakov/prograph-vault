# R-07 — choosing a direction after P2 closed

**Date:** 2026-06-18
**Context:** the P2 arc closed on 2026-06-17 — cases merged (atp `#195`), filter + held-out ran, handoff to arbiter ready.
**Sources:** `2026-06-17-p2-filter-results.md`, `2026-06-17-r07-task4-handoff-to-arbiter.md`, `2026-06-17-r07-matrix-and-spend.md`.

## TL;DR

1. **P2 won as outcome A:** on `code-review`, `claude_code > codex_cli` confirmed (held-out n=8, malformed≈0 → capability, not format). The ball is in arbiter's court (Task-4 A/B).
2. **But only half of the R-07 thesis is won.** This is *dominance on a single benchmark*, not a **crossover**. Task-dependent routing (a different winner per task type) among routable agents is still not demonstrated.
3. **The signal is narrow and one-directional:** both separating cases are *codex weaknesses* (`fp-deser-trusted` +0.50 sd=0; `distractor-singlefile` +0.59 ≈3.4σ), not a broad claude superiority. Meanwhile claude is ~80% slower (254s vs 142s/pass).
4. **A cheap risk hangs open:** codex `$` is unmeasured → the re-rank sees only quality. "All review → claude" = paying +80% latency for a gap that shows up only on two patterns.
5. **Recommendation — a sequence, not "either/or":** finish the cheap atp tasks in your own zone (codex token→$, +1–2 FP cases), hand Task-4 A/B to arbiter, aim for crossover (a second routable benchmark) as the next major front.

## Recommendation (sequence)

**A. Now, in parallel, your zone (cheap):** #3 + #2.
- codex `$` via a token→price lookup — without it a budget-aware re-rank is blind, and the decision "all review → claude" is economically unjustified (see TL;DR #4).
- +1–2 FP-discipline cases on other "safe-but-scary" patterns (not only `pickle`), so the signal doesn't hang on two points.

**B. Next gate — hand to arbiter, wait:** #1 Task-4 A/B.
- The necessary and cheap first live confirmation of the loop. But it confirms the *mechanism*, not the thesis: the re-rank merely prefers the better agent. There's no point chasing crossover before passing this gate.

**C. The strategic prize — after the gate:** #4.
- A second *routable* benchmark where **not** claude wins (candidate — req-extraction: in the matrix codex/deepseek are strong there), so the re-rank *switches* the agent by task type.
- Blocked on taxonomy: arbiter's `TaskType` enum = feature/bugfix/refactor/docs/review/research/test, with no mapping for req-extraction (P4). This is the main unsolved question of the whole endeavor.

**Principle:** don't open a new front today. Close the cheap #3/#2, hand off #1, keep #4 as the next big goal.
