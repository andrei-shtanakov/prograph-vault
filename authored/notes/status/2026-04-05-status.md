# Project Inventory: atp-platform, arbiter, Maestro

**Date:** 2026-04-05

## TL;DR

1. **atp-platform** — the most mature project (GA 1.0.0): 709 files, 301 tests, 7 CI pipelines, 88 documentation files. Practically zero tech debt. Active development (committed today).
2. **arbiter** — a stable MVP (0.1.0): Rust + Python, 32 code files, 238 Rust tests, zero TODO/FIXME. Development stopped ~2 months ago — the project reached its target state.
3. **Maestro** — an active beta (0.1.0): 67 Python files, 29 tests, dogfooding in progress. The most active by commits over the last week, but with no CI/CD.
4. **Links:** Maestro → Arbiter (via MCP route_task), Maestro → ATP (verification). Arbiter is autonomous. ATP is framework-agnostic.
5. **Main risk:** Maestro has no CI/CD, arbiter has no automated pipelines (only a Makefile). Only ATP has full-fledged CI.
