## 1. Qualcomm buries the compute under the memory: HBC architecture and the AI250 accelerator

**What happened:** Qualcomm disclosed details of its near-memory High-Bandwidth Compute (HBC) architecture for the AI200 and AI250 data-center accelerators. The company stacks LPDDR memory directly on top of the compute die, connecting them via through-silicon vias (TSV), and promises a 6x advantage in bandwidth per watt over HBM and 200x greater capacity than on-chip SRAM. The AI200 (LPDDR5X) delivers 768 GB of memory per card and 43 TB per rack; the AI250 on HBC Gen 1 ramps up to 133 TB/s per card.

**Technical details:** Racks are 160 kW, direct liquid cooling, PCIe for scale-up and Ethernet for scale-out. The focus is inference, not training: HBC optimizes the "memory wall" that pins LLM inference not to FLOPS but to bandwidth and capacity. The AI200 ships in 2026, the AI250 in 2027.

**My take:** This is a flanking move, and a smart one. While everyone fights over HBM — scarce and expensive — Qualcomm bets on cheap LPDDR and clever packaging — a classic "if you can't win head-on, change the rules" play. Betting specifically on inference is also right: the model-serving market grows faster than the training market, and there the bandwidth-per-watt economics decide everything. But I'm skeptical about execution — Qualcomm has announced data-center hardware many times and retreated, and near-memory computing has historically stumbled over software and toolchains. If the ecosystem (compilers, runtimes) doesn't catch up, 133 TB/s will remain a pretty slide. What to watch is not the numbers on paper but the real deployments at hyperscalers in 2027.

## 2. Citrix patches six holes in NetScaler, including a new "HTTP/2 Bomb" echoing CitrixBleed

**What happened:** On June 30, Citrix issued a security bulletin with patches for six vulnerabilities in NetScaler ADC and Gateway. Among them is CVE-2026-8451 (CVSS 8.8): a memory overread when the device operates as a SAML Identity Provider, which draws direct associations for researchers with CitrixBleed. Also closed are CVE-2026-10816 (unauthenticated arbitrary file read when the management interface is open) and a new attack class, "HTTP/2 Bomb" (CVE-2026-13474) — a DoS via malformed HTTP/2 requests.

**Technical details:** Fixed in versions 14.1-72.61 and 13.1-63.18 (plus FIPS builds). To fully close HTTP/2 Bomb, one patch isn't enough — you need to manually set the Http2SmallWndTimeout parameter to 30 seconds on devices without HTTP Strict Profiles. There are no signs of exploitation in the wild yet.

**My take:** "Not yet exploited" on NetScaler is a phrase with a shelf life of a couple of weeks. The history of CitrixBleed and CitrixBleed 2 showed that Citrix edge devices are a favorite skeleton key of ransomware groups, and a memory overread in the SAML IdP smells exactly of that same class of session-token leak. Separately infuriating is the pattern "install the patch, then also fix the config by hand" — half the admins will never learn about Http2SmallWndTimeout, and HTTP/2 Bomb will stay alive. The real lesson here isn't "apply the update" but "remove the NetScaler management interface from the internet altogether" — CVE-2026-10816 reads files exactly where the device is exposed outward. If your ADC is reachable from outside for management — you're already in the risk zone, patch or no patch.

## 3. Azure CLI under fire: 81 million attempts, 78 breached accounts, and MFA bypass via ROPC

**What happened:** Huntress disclosed a large-scale password-spraying campaign against Microsoft 365: from June 12 to 26, the attackers made more than 81 million login attempts and compromised at least 78 accounts across 64 organizations. The attack ran through Azure CLI using login-password pairs from old leaks. The source was the IPv6 range of LSHIY LLC (AS32167).

**Technical details:** The key to the attack is the outdated OAuth flow Resource Owner Password Credentials (ROPC), which bypasses poorly configured Conditional Access Policies. Since CAPs usually require MFA at the authorization endpoint, and ROPC doesn't go there, a token is issued without an MFA challenge. Huntress records a 155x increase in password-spraying: on average 1,964 failed logins per tenant per month.

**My take:** This isn't about a clever exploit but about the fact that legacy authentication protocols are still on by default, and that's a disgrace to the industry. ROPC should have died years ago — Microsoft deprecates it itself but doesn't disable it, so as not to break someone's old script. As a result, MFA — which everyone paid for and treats as a security checkbox — is bypassed not by hacking but simply by walking through a door someone forgot to nail shut. The practical takeaway is harsh: disable ROPC and legacy authentication at the tenant level right now, without relying on Conditional Access to cover you. And stop measuring security by the fact that "we have MFA enabled" — what matters is which flows bypass it.

## 5. Rust in the Linux kernel — not an experiment but a given; Bcachefs brings Rust into the driver

**What happened:** Following discussions at the Linux Plumbers Conference, it was officially recorded: the "experiment" with the Rust language in the Linux kernel is over and deemed a success — Rust stays in the kernel seriously and for the long term. In parallel, the Bcachefs file system, which already uses Rust in userspace utilities, announced plans to include Rust code directly in the kernel driver starting with the v1.38.7 release.

**Technical details:** This is about the shift from "trial" infrastructure status (Rust abstractions over C subsystems) to full-fledged support you can rely on when writing new drivers. Bcachefs will become one of the first major consumers of Rust in the FS driver itself, not just in accompanying tools.

**My take:** This is a quiet but historic shift. The "does the kernel need Rust" argument dragged on for years, with maintainers walking away and public conflicts, and now it's settled de facto — memory safety beat C's inertia. For systems developers this is a signal: investing in Rust for kernel-space is no longer a risky bet but a career necessity. But caution is warranted: "Rust stays" doesn't mean "C leaves" — the kernel will remain hybrid for decades, and the main pain will be at the boundary of the two languages, where the nastiest bugs live. The story of Bcachefs, which already has a strained relationship with Torvalds, will yet show how smoothly Rust drivers pass review in practice. The direction is correct, but the real test is not declarations but the first serious Rust driver that survives a couple of LTS cycles without a rewrite.
