But strategically NVIDIA is closing the entire Physical AI stack onto itself (simulation → world model → VLA → GPU), and this lock-in is far more durable than any single SoTA LLM. If you're building something beyond pure text, this direction is worth watching now — the next platform is being laid down here, not another chatbot.

## 5. MCP prepares its biggest overhaul: away with sessions, new auth, an extensions framework

**What happened:** The maintainers of the Model Context Protocol published a release candidate of the MCP 2026-07-28 specification, calling it the biggest revision since launch. The changes are radical: sessions are removed, the initialization handshake is stripped out, three core base features are deprecated, authorization is completely rewritten and an extensions framework is introduced. The final spec is expected on July 28; a 10-week window is given to SDK and client maintainers for validation. By this point MCP has surpassed 200+ server implementations.

**Architecture/Details:** Dropping sessions makes the protocol friendly to stateless/serverless and horizontal scaling; a simplified handshake and a new auth model are a bid for production-grade; the extensions framework sets a trajectory for development without new breaking core revisions.

**My take:** MCP is going through its "cleanup before 1.0" — and removing sessions along with rewriting auth clearly signals that the protocol is hardening for enterprise and production, not for local toolchains. Stateless-friendliness is critical for real deployments in serverless and behind a load balancer. This is the maturing of the protocol layer — and, as I noted about ACP a week ago, whoever wins the standard wins the ecosystem lock-in. The honest skepticism: "biggest revision" = breaking changes = migration pain for 200+ servers, and frequent breaking changes before 1.0 erode trust. For those building tools: don't hard-wire yourself to the current spec, and budget for a migration by July 28 now.
